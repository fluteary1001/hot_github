# 历史官方热点功能设计说明书

> 版本：1.0.0 | 日期：2026-05-18

---

## 1. 功能概述

### 1.1 功能名称

历史官方热点 (Historical Trending)

### 1.2 功能定位

在现有 GitHub Manager Web 项目基础上，增加历史热点数据的自动采集、存储、展示与分析功能。实现"定时采集 → 自动下载 → AI分析生成报告"的完整闭环。

### 1.3 核心价值

| 能力 | 描述 |
|---|---|
| 历史数据存储 | 每日/每周/每月热点数据持久化到数据库 |
| 树状浏览 | 年 → 月 → 周 → 日 层级结构展示历史热点 |
| 定时采集 | 晚上10点自动采集热点数据 |
| 自动下载 | 采集后自动下载 TOP N 项目 |
| AI分析报告 | 凌晨自动生成设计说明书、使用说明书、价值点分析 HTML 报告 |
| 快捷访问 | 项目管理页面增加报告快捷链接 |

## 3. 数据库设计

### 3.1 trending_history 表（热点历史记录）

```sql
CREATE TABLE IF NOT EXISTS trending_history (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    period_type TEXT NOT NULL,           -- 周期类型: 'daily'/'weekly'/'monthly'
    period_value TEXT NOT NULL,          -- 周期值: '2026-05-18'/'2026-W20'/'2026-05'
    year INTEGER NOT NULL,               -- 年份: 2026
    month INTEGER NOT NULL,              -- 月份: 1-12
    week INTEGER,                        -- 周数: 1-53 (仅weekly)
    day INTEGER,                         -- 日期: 1-31 (仅daily)
    rank INTEGER NOT NULL,               -- 排名: 1-N
    project_name TEXT NOT NULL,          -- 项目名称
    full_name TEXT NOT NULL,             -- 完整名称: owner/repo
    html_url TEXT,                       -- GitHub URL
    description TEXT,                    -- 项目描述
    stars INTEGER DEFAULT 0,             -- Star 数
    stars_change INTEGER DEFAULT 0,      -- Star 增长数
    language TEXT,                       -- 编程语言
    clone_url TEXT,                      -- Git clone URL
    collected_at TEXT NOT NULL,          -- 采集时间
    downloaded INTEGER DEFAULT 0,        -- 是否已下载: 0/1
    project_id INTEGER,                  -- 关联 projects 表 ID
    UNIQUE(period_type, period_value, full_name, rank)
);

CREATE INDEX idx_trending_history_period ON trending_history(period_type, period_value);
CREATE INDEX idx_trending_history_year_month ON trending_history(year, month);
```

### 3.2 trending_reports 表（AI分析报告记录）

```sql
CREATE TABLE IF NOT EXISTS trending_reports (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    history_id INTEGER NOT NULL,         -- 关联 trending_history.id
    project_id INTEGER,                  -- 关联 projects.id
    project_name TEXT NOT NULL,          -- 项目名称
    design_doc_path TEXT,                -- 设计说明书路径
    usage_doc_path TEXT,                 -- 使用说明书路径
    value_doc_path TEXT,                 -- 价值点分析路径
    design_html_path TEXT,               -- 设计说明书 HTML 路径
    usage_html_path TEXT,                -- 使用说明书 HTML 路径
    value_html_path TEXT,                -- 价值点分析 HTML 路径
    status TEXT DEFAULT 'pending',       -- 状态: pending/processing/completed/failed
    error_message TEXT,                  -- 错误信息
    created_at TEXT NOT NULL,            -- 创建时间
    updated_at TEXT,                     -- 更新时间
    retry_count INTEGER DEFAULT 0,       -- 重试次数
    FOREIGN KEY (history_id) REFERENCES trending_history(id)
);

CREATE INDEX idx_trending_reports_status ON trending_reports(status);
CREATE INDEX idx_trending_reports_history ON trending_reports(history_id);
```

## 5. 定时任务设计

### 5.1 任务调度配置

```python
# backend/app/core/scheduler.py

from apscheduler.schedulers.background import BackgroundScheduler
from apscheduler.triggers.cron import CronTrigger
from datetime import datetime

class TrendingScheduler:
    """热点采集调度器"""

    def __init__(self):
        self.scheduler = BackgroundScheduler()
        self._setup_jobs()

    def _setup_jobs(self):
        """配置定时任务"""

        # 每天 22:00 采集日热点
        self.scheduler.add_job(
            self.collect_daily_trending,
            CronTrigger(hour=22, minute=0),
            id='daily_trending',
            name='每日热点采集'
        )

        # 每周一 22:00 采集周热点
        self.scheduler.add_job(
            self.collect_weekly_trending,
            CronTrigger(day_of_week='mon', hour=22, minute=0),
            id='weekly_trending',
            name='每周热点采集'
        )

        # 每月1日 22:00 采集月热点
        self.scheduler.add_job(
            self.collect_monthly_trending,
            CronTrigger(day=1, hour=22, minute=0),
            id='monthly_trending',
            name='每月热点采集'
        )

        # 凌晨 00:30 开始自动下载和分析
        self.scheduler.add_job(
            self.process_auto_download_and_analysis,
            CronTrigger(hour=0, minute=30),
            id='auto_analysis',
            name='自动下载与分析'
        )

        # 失败重试任务，每小时检查一次
        self.scheduler.add_job(
            self.retry_failed_analysis,
            CronTrigger(hour='*', minute=0),
            id='retry_analysis',
            name='失败重试'
        )
```

### 5.2 任务执行流程

```
┌─────────────────────────────────────────────────────────────┐
│                    22:00 热点采集                            │
├─────────────────────────────────────────────────────────────┤
│  1. 调用 TrendingScraper.get_trending(since='daily')        │
│  2. 遍历 TOP N 项目，保存到 trending_history 表              │
│  3. 记录采集时间、周期信息                                    │
│  4. 更新 scheduler_tasks 表状态                              │
└─────────────────────────────────────────────────────────────┘
                              │
                              ▼
┌─────────────────────────────────────────────────────────────┐
│                    22:00+ 立即下载                           │
├─────────────────────────────────────────────────────────────┤
│  1. 查询本次采集的 TOP N 项目                                 │
│  2. 检查是否已下载（projects 表中存在）                       │
│  3. 未下载 → 执行 git clone                                  │
│  4. 已下载 → 执行 git pull 更新                              │
│  5. 更新 trending_history.downloaded = 1                    │
│  6. 关联 project_id                                          │
└─────────────────────────────────────────────────────────────┘
                              │
                              ▼
┌─────────────────────────────────────────────────────────────┐
│                    00:30 AI分析生成                           │
├─────────────────────────────────────────────────────────────┤
│  1. 检查所有下载任务是否完成                                   │
│  2. 查询今日采集且已下载的项目                                 │
│  3. 检查是否已生成报告（trending_reports 表）                 │
│  4. 串行执行 AI 分析：                                        │
│     for project in projects:                                 │
│         - 生成设计说明书 HTML                                  │
│         - 生成使用说明书 HTML                                  │
│         - 生成价值点分析 HTML                                  │
│         - 记录到 trending_reports 表                          │
│         - 失败则继续下一个                                    │
│  5. 所有完成后，统计失败项目                                   │
│  6. 有失败 → 1小时后重试                                      │
└─────────────────────────────────────────────────────────────┘
```

### 5.3 失败重试机制

```python
class AnalysisRetryManager:
    """分析失败重试管理"""

    MAX_RETRY = 3
    RETRY_INTERVAL = 3600  # 1小时

    def get_failed_projects(self) -> list:
        """获取需要重试的项目"""
        return db.query("""
            SELECT * FROM trending_reports
            WHERE status = 'failed'
            AND retry_count < ?
            AND datetime(updated_at) < datetime('now', '-1 hour')
        """, (self.MAX_RETRY,))

    def retry_analysis(self):
        """重试失败的分析任务"""
        failed = self.get_failed_projects()
        for report in failed:
            try:
                self.run_analysis(report)
                report.status = 'completed'
            except Exception as e:
                report.retry_count += 1
                report.error_message = str(e)
                if report.retry_count >= self.MAX_RETRY:
                    report.status = 'exhausted'
```

## 11. 注意事项

### 11.1 性能考虑

- 树状数据懒加载，避免一次性加载过多数据
- AI 分析串行执行，避免并发导致资源耗尽
- 定时任务使用后台线程，不阻塞主进程

### 11.2 容错处理

- 采集失败记录日志，不影响其他任务
- 下载失败标记状态，支持手动重试
- 分析失败自动重试，超过次数标记为失败

### 11.3 数据一致性

- 使用 UNIQUE 约束避免重复采集
- 事务保证热点数据和报告数据一致性
- 定时任务状态持久化到数据库

### 12.1 周数计算

```python
from datetime import datetime

def get_week_number(date: datetime) -> tuple:
    """获取日期的年周数"""
    iso_calendar = date.isocalendar()
    return iso_calendar[0], iso_calendar[1]  # year, week
```
