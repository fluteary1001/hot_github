```javascript
function getFileUrl(projectId: number, path: string): string {
  if (!path || projectId <= 0) return '#'
  const fileName = path.split('/').pop() || path.split('\\').pop()
  const encodedFileName = encodeURIComponent(fileName || '')
  return `/api/docs/file/${projectId}/${encodedFileName}`
}
```

function getFileUrl(projectId: number, path: string): string {
  if (!path || projectId <= 0) return '#'
  // 转换本地路径为 API 路径
  const fileName = path.split('/').pop() || path.split('\\').pop()
  // 对文件名进行 URL 编码，确保中文字符正确传递
  const encodedFileName = encodeURIComponent(fileName || '')
  return `/api/docs/file/${projectId}/${encodedFileName}`
}
```

    matching_files = list(docs_dir.glob(f"{base_name}*.{extension}"))
    # 匹配结果: ["academic-research-skills-设计说明书-20260519.html"]
    
    if matching_files:
        # 按修改时间排序，取最新的
        matching_files.sort(key=lambda f: f.stat().st_mtime, reverse=True)
        doc_path = matching_files[0]
        # doc_path = "../github_projects/01-docs/academic-research-skills-设计说明书-20260519.html"
```

**匹配结果**: 成功匹配到 `academic-research-skills-设计说明书-20260519.html`

#### 4.5 返回 HTML 内容

```python
# 确定文件类型
suffix = doc_path.suffix.lower()  # ".html"

if suffix in ['.html', '.htm']:
    # 读取文件内容
    with open(doc_path, 'r', encoding='utf-8') as f:
        content = f.read()
    
    # 返回 HTML 响应（内嵌显示）
    response = Response(
        content=content,
        media_type='text/html',
        headers={'Content-Disposition': 'inline'}
    )
    return response
```

## 4. 关键技术点

### 4.1 URL 编码

**问题**: 文件名包含中文字符，直接拼接 URL会导致乱码

**解决方案**: 使用 `encodeURIComponent` 编码

```javascript
const fileName = "academic-research-skills-设计说明书.html"
const encodedFileName = encodeURIComponent(fileName)
// 结果: "academic-research-skills-%E8%AE%BE%E8%AE%A1%E8%AF%B4%E6%98%8E%E4%B9%A6.html"
```

| 字符 | UTF-8 编码 | URL 编码 |
|---|---|---|
| 设 | E8 AE BE | %E8%AE%BE |
| 计 | E8 AE A1 | %E8%AE%A1 |
| 说 | E8 AF B4 | %E8%AF%B4 |
| 明 | E6 98 8E | %E6%98%8E |
| 书 | E4 B9 A6 | %E4%B9%A6 |

### 4.2 文件名模糊匹配

**问题**: 数据库存储的文件名与实际文件名不一致

**匹配逻辑**:
```
输入文件名: academic-research-skills-设计说明书.html
           ↓
     去掉扩展名
           ↓
基础名: academic-research-skills-设计说明书
           ↓
     glob 模式匹配
           ↓
匹配结果: academic-research-skills-设计说明书-20260519.html
           ↓
     按时间排序取最新
           ↓
最终文件: academic-research-skills-设计说明书-20260519.html
```

**glob 模式**: `{base_name}*.{extension}`
- `*` 匹配任意字符（包括日期后缀）
- 例如 `academic-research-skills-设计说明书-*.html`

## 6. 潜在问题与解决方案

### 6.1 URL 编码缺失

**症状**: 点击链接返回 `{"detail":"文档不存在"}`

**原因**: 中文字符未编码，FastAPI 无法正确解析路径参数

**解决方案**: 前端使用 `encodeURIComponent` 编码文件名

### 6.2 文件名不匹配

**症状**: 返回 `{"detail":"文档不存在"}`（即使 URL 编码正确）

**原因**: 数据库存储的路径与实际文件名有差异（日期后缀）

**解决方案**: 后端增加模糊匹配，使用 glob 模式查找前缀匹配的文件

### 6.3 project_id 错误

**症状**: 返回 `{"detail":"项目不存在"}`

**原因**: 前端硬编码 `project_id=0`

**解决方案**: 使用 `project.project_id` 从数据中获取正确的 ID
