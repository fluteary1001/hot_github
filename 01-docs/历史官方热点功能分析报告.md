# 历史官方热点功能分析报告

## 一、功能概述

"历史官方热点"功能是一个完整的 GitHub Trending 数据采集、存储、展示和分析系统。它能够定期自动采集 GitHub 官方 Trending 页面的热点项目数据，保存历史记录，并提供可视化界面查看历史热点数据，同时支持自动下载和 AI 分析。

## 二、核心模块

### 2.1 后端模块

| 文件路径 | 功能描述 |
|---------|---------|
| `backend/app/core/trending_scraper.py` | GitHub Trending 页面爬虫 |
| `backend/app/core/history_manager.py` | 历史热点数据管理 |
| `backend/app/core/database.py` | 数据库操作（含历史热点相关表） |
| `backend/app/core/scheduler.py` | 定时任务调度器 |
| `backend/app/api/history_trending.py` | 历史热点 API 路由 |
| `backend/app/api/trending.py` | 热点项目 API 路由 |

### 2.2 前端模块

| 文件路径 | 功能描述 |
|---------|---------|
| `frontend/src/pages/HistoryTrendingPage.vue` | 历史热点页面组件 |
| `frontend/src/api/history.ts` | 历史热点 API 接口 |

## 五、API 接口

### 5.1 历史热点 API（/api/history）

| 接口 | 方法 | 说明 |
|------|------|------|
| `/tree` | GET | 获取历史热点树状结构（年-月-周-日） |
| `/years` | GET | 获取有数据的年份列表 |
| `/months` | GET | 获取指定年份的月份数据 |
| `/weeks` | GET | 获取指定月份的周数据 |
| `/days` | GET | 获取指定周或月份的日数据 |
| `/detail` | GET | 获取指定周期的热点详情 |
| `/collect` | POST | 手动触发热点采集 |
| `/scheduler/status` | GET | 获取定时任务状态 |

### 5.2 热点项目 API（/api/trending）

| 接口 | 方法 | 说明 |
|------|------|------|
| `/official` | GET | 获取 GitHub 官方 Trending |
| `/stars` | GET | 获取高星项目（按主题筛选） |
| `/categories` | GET | 获取所有主题分类 |

## 六、定时任务

调度器使用 APScheduler 实现，配置如下：

| 任务 ID | 执行时间 | 功能 |
|---------|---------|------|
| daily_trending | 每天 22:00 | 采集每日热点 |
| weekly_trending | 每周一 22:00 | 采集每周热点 |
| monthly_trending | 每月1日 22:00 | 采集每月热点 |
| auto_analysis | 每天 00:30 | 自动下载和分析 |
| retry_analysis | 每小时整点 | 重试失败的分析任务 |

## 七、核心类说明

### 7.1 TrendingScraper（爬虫类）

**位置**: `backend/app/core/trending_scraper.py`

**功能**: 爬取 GitHub Trending 页面

**主要方法**:
- `get_trending(since, language, limit)`: 获取热点列表
- `_parse_trending_page(html, limit)`: 解析 HTML 页面
- `_parse_repo_article(article)`: 解析单个项目条目

**数据来源**: `https://github.com/trending`

**支持参数**:
- `since`: 时间范围（daily/weekly/monthly）
- `language`: 编程语言筛选
- `limit`: 最大返回数量

### 7.2 HistoryManager（历史管理类）

**位置**: `backend/app/core/history_manager.py`

**功能**: 管理历史热点数据

**主要方法**:
- `get_tree_data()`: 获取完整树状结构
- `get_period_detail(period_type, period_value)`: 获取周期详情
- `get_today_period_value()`: 获取今日周期值
- `get_this_week_period_value()`: 获取本周周期值
- `get_this_month_period_value()`: 获取本月周期值

### 7.3 TrendingScheduler（调度器类）

**位置**: `backend/app/core/scheduler.py`

**功能**: 定时任务调度

**主要方法**:
- `start()`: 启动调度器
- `shutdown()`: 关闭调度器
- `collect_daily_trending()`: 采集每日热点
- `collect_weekly_trending()`: 采集每周热点
- `collect_monthly_trending()`: 采集每月热点
- `manual_collect(period_type)`: 手动触发采集
- `_auto_download_trending(period_value)`: 自动下载热点项目
- `_run_analysis_for_history(history)`: 执行 AI 分析

## 九、配置参数

相关配置在 `backend/app/config.py` 中：

| 参数名 | 说明 |
|--------|------|
| `GITHUB_TOKEN` | GitHub API Token |
| `GITHUB_API_TIMEOUT` | API 超时时间 |
| `TRENDING_OFFICIAL_LIMIT` | 官方热点采集数量限制 |
| `DOWNLOAD_PATH` | 项目下载路径 |
| `HISTORY_TRENDING_AUTO_DOWNLOAD` | 是否自动下载热点项目 |
| `HISTORY_TRENDING_AUTO_ANALYSIS` | 是否自动分析热点项目 |
| `HISTORY_TRENDING_MAX_RETRY` | 分析失败最大重试次数 |

## 十、技术栈

- **后端**: FastAPI + SQLite + APScheduler + BeautifulSoup4
- **前端**: Vue 3 + TypeScript + Element Plus
- **爬虫**: requests + BeautifulSoup4
- **AI 分析**: Claude API（通过 WebDocGenerator）

## 十二、总结

"历史官方热点"功能是一个完整的 GitHub Trending 数据管理系统，具备以下特点：

1. **自动化采集**: 定时自动采集 GitHub 官方 Trending 数据
2. **历史数据存储**: 支持按日/周/月维度存储历史热点
3. **可视化展示**: 树形结构展示历史数据，支持详情查看
4. **自动下载**: 可配置自动下载热点项目到本地
5. **AI 分析**: 可配置自动生成项目分析报告
6. **失败重试**: 支持分析任务失败自动重试
7. **手动触发**: 支持手动触发采集操作
