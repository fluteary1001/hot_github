> 版本：2.0.0 | 日期：2026-05-17

## 1. 项目概述

### 1.1 项目名称

GitHub Manager Web — 开源项目银河管理站

### 1.2 项目定位

面向开发者的一站式 GitHub 开源项目管理平台，提供项目搜索、下载、趋势追踪与 AI 智能文档生成功能。以"深空观测站"为视觉主题，打造沉浸式暗色系管理体验。

### 1.3 核心价值

| 能力 | 描述 |
|---|---|
| 智能搜索 | 关键词搜索 + URL 直达，快速定位 GitHub 仓库 |
| 一键下载 | 后台 git clone，实时进度追踪 |
| 趋势洞察 | 官方 Trending 爬取 + 高星项目分类浏览 |
| 项目管理 | 下载项目 CRUD、本地导入、git pull 更新 |
| AI 文档生成 | 调用 Claude 生成设计说明书、使用指南、价值分析（MD/HTML/PPTX 三阶段） |

## 4. 数据库设计

### 4.1 数据库选型

SQLite3 — 轻量、零配置、单文件存储，适合单机桌面级应用。

数据库文件位置：`~/github_projects/projects.db`

### 4.2 表结构

#### projects 表

| 字段 | 类型 | 约束 | 说明 |
|---|---|---|---|
| id | INTEGER | PRIMARY KEY AUTOINCREMENT | 主键 |
| name | TEXT | NOT NULL | 项目名称 |
| full_name | TEXT | UNIQUE | 完整名称（owner/repo） |
| html_url | TEXT | — | GitHub 页面 URL |
| description | TEXT | — | 项目描述 |
| stars | INTEGER | — | Star 数 |
| forks | INTEGER | — | Fork 数 |
| language | TEXT | — | 主要编程语言 |
| topics | TEXT | — | 话题标签（JSON 数组） |
| local_path | TEXT | — | 本地文件系统路径 |
| shortcut_path | TEXT | — | 快捷方式路径（预留） |
| clone_url | TEXT | — | Git clone URL |
| created_at | TEXT | — | 仓库创建时间 |
| updated_at | TEXT | — | 仓库最后推送时间 |
| downloaded_at | TEXT | — | 本地下载时间 |
| usage_guide | TEXT | — | 使用指南（可选） |
| competitors | TEXT | — | 竞品信息（可选） |
| is_manual | INTEGER | — | 是否手动导入（0/1） |

### 4.3 关键操作

- **Upsert 逻辑**：插入时若 `full_name` 已存在，则更新 stars、forks、description、language、topics 字段
- **级联删除**：删除项目记录时，可选同时删除本地文件（`delete_files` 参数）
- **全文搜索**：通过 SQL `LIKE` 对 name、full_name、description 进行模糊匹配

## 5. API 接口设计

### 5.1 认证模块 `/api/auth`

| 方法 | 路径 | 说明 | 认证 |
|---|---|---|---|
| POST | `/api/auth/login` | 用户登录，返回 JWT | 否 |
| POST | `/api/auth/logout` | 用户登出（前端清除 token） | 是 |
| GET | `/api/auth/me` | 获取当前用户信息 | 是 |

**登录流程**：

```
客户端 POST {username, password}
    → 服务端校验 USERS 字典
    → 签发 JWT (HS256, 24h过期)
    → 返回 {access_token, token_type, username}
```

### 5.2 搜索与下载模块 `/api/search`

| 方法 | 路径 | 说明 |
|---|---|---|
| POST | `/api/search/` | 关键词搜索 GitHub 仓库 |
| POST | `/api/search/url` | 通过 GitHub URL 搜索 |
| POST | `/api/search/download` | 启动后台 git clone 任务 |
| GET | `/api/search/download/{task_id}` | 轮询下载任务状态 |
| GET | `/api/search/test-connection` | 测试 GitHub 连接 |

**下载流程**：

```
客户端 POST /api/search/download {clone_url, ...}
    → 后端创建后台任务，执行 git clone
    → 返回 task_id
    → 客户端轮询 GET /api/search/download/{task_id}
    → 完成后写入 projects 表
```

### 5.3 项目管理模块 `/api/projects`

| 方法 | 路径 | 说明 |
|---|---|---|
| GET | `/api/projects/` | 项目列表（支持 keyword 过滤） |
| GET | `/api/projects/{id}` | 项目详情 |
| POST | `/api/projects/import` | 导入本地目录 |
| PUT | `/api/projects/{id}` | 更新项目元数据 |
| DELETE | `/api/projects/{id}` | 删除项目（可选 delete_files） |
| POST | `/api/projects/{id}/update` | 执行 git pull 更新 |

### 5.4 趋势模块 `/api/trending`

| 方法 | 路径 | 说明 |
|---|---|---|
| GET | `/api/trending/official` | 爬取 GitHub 官方 Trending 页 |
| GET | `/api/trending/stars` | 按分类获取高星项目 |
| GET | `/api/trending/categories` | 获取可用分类列表 |

### 5.5 AI 文档生成模块 `/api/docs`

| 方法 | 路径 | 说明 |
|---|---|---|
| POST | `/api/docs/generate/{project_id}` | 启动 AI 文档生成任务 |
| GET | `/api/docs/status/{task_id}` | 查询生成任务状态 |
| GET | `/api/docs/{project_id}` | 列出项目已生成文档 |
| GET | `/api/docs/file/{project_id}/{doc_name}` | 下载文档文件 |
| DELETE | `/api/docs/task/{task_id}` | 取消/清理生成任务 |

**文档生成三阶段**：

```
Phase 1: 生成 Markdown
  ├── 设计说明书.md
  ├── 使用指南.md
  └── 价值分析.md

Phase 2: 生成 HTML
  ├── 设计说明书.html
  ├── 使用指南.html
  └── 价值分析.html

Phase 3: 生成 PPTX
  ├── 设计说明书.pptx
  ├── 使用指南.pptx
  └── 价值分析.pptx
```

### 5.6 系统配置模块 `/api/config`

| 方法 | 路径 | 说明 |
|---|---|---|
| GET | `/api/config` | 获取当前配置 |
| PUT | `/api/config` | 更新并持久化配置 |
| POST | `/api/config/test-github` | 测试 GitHub 连接 |

### 5.7 WebSocket

| 路径 | 说明 |
|---|---|
| `/ws/download/{task_id}` | 下载任务实时进度 |

### 5.8 其他

| 方法 | 路径 | 说明 |
|---|---|---|

## 7. 核心模块设计

### 7.1 认证系统

```
┌──────────┐    POST /login     ┌──────────┐    校验     ┌──────────┐
│  LoginPage│ ───────────────→ │ auth.py  │ ────────→ │ USERS字典 │
└──────────┘                   └────┬─────┘            └──────────┘
                                    │
                               签发 JWT
                                    │
                               ┌────▼─────┐
                               │  Pinia   │
                               │ authStore│
                               └──────────┘
                                    │
                          Axios 拦截器自动附加
                          Authorization: Bearer <token>
```

- 算法：HS256
- 过期时间：24 小时
- 存储位置：前端 `localStorage`
- 路由守卫：未认证自动重定向至 `/login`
- 401 响应：自动清除 token 并跳转登录页

### 7.2 搜索与下载

```
用户输入关键词
    → 后端调用 GitHub Search API: GET /search/repositories
    → 返回仓库列表（名称、描述、Star、语言等）

用户点击下载
    → POST /api/search/download
    → 后端 asyncio.create_task(git clone)
    → 返回 task_id
    → 前端轮询 GET /api/search/download/{task_id}
    → clone 完成后：
        1. 解析仓库元数据
        2. UPSERT 到 projects 表
        3. 返回完成状态
```

### 7.3 趋势爬取

```
官方 Trending:
    GET /api/trending/official?since=daily
    → requests.get("https://github.com/trending")
    → BeautifulSoup4 解析 HTML
    → 提取：仓库名、描述、Star、今日新增、语言

高星项目:
    GET /api/trending/stars?category=ai&min_stars=1000
    → GitHub Search API: q="topic:ai stars:>1000"
    → 按 Star 排序，计算日均 Star 增长率
```

### 7.4 AI 文档生成

```
用户触发生成
    → POST /api/docs/generate/{project_id}
    → 读取项目本地目录结构和关键文件
    → 构建分析上下文（目录树 + README + 主要源码）
    → 三阶段顺序生成：

    Phase 1: Markdown
      → 调用 Claude（CLI 或 API）
      → 逐个生成：设计说明书.md → 使用指南.md → 价值分析.md
      → 每步回调更新进度

    Phase 2: HTML
      → 将 Markdown 转换为精美 HTML
      → 逐个生成：设计说明书.html → 使用指南.html → 价值分析.html

    Phase 3: PPTX
      → 基于 HTML 内容生成 PowerPoint
      → 逐个生成：设计说明书.pptx → 使用指南.pptx → 价值分析.pptx

    完成后文件保存至项目本地目录
```

**Claude 调用方式**（二选一）：

| 方式 | 实现 | 适用场景 |
|---|---|---|
| CLI 模式 | `subprocess.run(["claude", "-p", "--print", prompt])` | 本地安装了 Claude Code CLI |
| API 模式 | `httpx.post(CLAUDE_BASE_URL, headers, json)` | 直接调用 Anthropic API |

### 7.5 实时进度推送

```
┌───────────┐    创建任务    ┌──────────────┐
│  前端页面  │ ───────────→ │  后端任务管理  │
└─────┬─────┘              └──────┬───────┘
      │                           │
      │  轮询 / WebSocket         │  回调更新进度
      │◀─────────────────────────│
      │                           │
  更新进度条/状态文字          ProgressManager
```

- 当前前端主要使用 HTTP 轮询获取下载进度
- WebSocket 通道已建立，可用于文档生成进度推送

## 8. 配置管理

### 8.1 配置来源优先级

```
环境变量 (.env) → config.json → 默认值
```

### 8.2 核心配置项

| 配置项 | 环境变量 | 默认值 | 说明 |
|---|---|---|---|
| GitHub Token | `GITHUB_TOKEN` | 空 | GitHub API 认证，提升请求限额 |
| Claude API Key | `CLAUDE_API_KEY` | 空 | AI 文档生成密钥 |

通过 `/api/config` GET/PUT 接口，前端可读取和修改配置。修改后的值持久化到项目根目录的 `config.json` 文件。

- **色彩体系**：CSS 变量驱动，深蓝/深紫背景 + 青色/品红点缀

## 10. 安全设计

### 10.1 当前实现

| 项目 | 实现 |
|---|---|
| 认证 | JWT (HS256)，24h 过期 |
| 密码存储 | 硬编码明文字典 `{"admin": "admin123"}` |
| CORS | 白名单限制 |
| API 保护 | `verify_token` 依赖项保护所有业务路由 |

### 10.2 已知风险

| 风险 | 严重程度 | 说明 |
|---|---|---|
| JWT 密钥硬编码默认值 | 高 | 生产环境必须替换 |
| 密码明文存储 | 高 | passlib 已引入但未实际使用 |
| 无用户注册机制 | 中 | 仅支持单一管理员账户 |
| GitHub Token 可通过 API 读取 | 中 | 配置接口返回敏感信息 |
| 无速率限制 | 中 | API 可被暴力请求 |

## 11. 部署方案

### 11.1 开发环境

```bash
# 后端
cd backend
pip install -r requirements.txt
python run.py          # 启动 Uvicorn (0.0.0.0:8000, 热重载)

# 前端
cd frontend
npm install
npm run dev            # 启动 Vite (localhost:5173, 代理到后端)
```

### 11.2 生产构建

```bash
# 前端构建
cd frontend
npm run build          # vue-tsc 类型检查 + vite build → dist/

# 后端
cd backend
python run.py          # 生产环境建议关闭 reload
```

## 13. 数据流图

### 13.1 项目搜索与下载

```
[用户] → 输入关键词 → [SearchPage]
                            │
                    POST /api/search
                            │
                    [github_api.py] → GitHub REST API
                            │
                    返回仓库列表 → [SearchPage] 展示卡片
                            │
                 用户点击下载 → POST /api/search/download
                            │
                    [downloader.py] → git clone (后台任务)
                            │
                    轮询进度 → GET /api/search/download/{id}
                            │
                    完成后 → [database.py] UPSERT projects 表
                            │
                    [ProjectsPage] 展示已下载项目
```

### 13.2 AI 文档生成

```
[用户] → 选择项目 → 触发生成 → [ProjectsPage]
                                    │
                        POST /api/docs/generate/{id}
                                    │
                        [doc_generator.py]
                          ├── 读取项目目录结构
                          ├── 提取关键文件内容
                          ├── 构建 Prompt
                          │
                          ├── Phase 1: Claude → 3个 .md 文件
                          ├── Phase 2: Claude → 3个 .html 文件
                          └── Phase 3: Claude → 3个 .pptx 文件
                                    │
                        轮询/Ws推送进度 → 前端更新进度条
                                    │
                        完成 → GET /api/docs/file/{id}/{name} → 下载
```

## 14. 错误处理

| 场景 | 处理方式 |
|---|---|
| GitHub API 限流 | 返回 429 提示，建议配置 GitHub Token |
| git clone 失败 | 记录日志，任务状态标记为 failed，前端展示错误信息 |
| Claude API 调用失败 | 中断当前阶段，标记任务失败，保留已生成文件 |
| JWT 过期 | 前端 401 拦截器自动跳转登录页 |
| 网络异常 | Axios 拦截器统一提示，前端展示友好错误信息 |
| 数据库锁 | SQLite WAL 模式，重试机制 |
