# GitHub Manager Web 配置参数说明

## 概述

所有配置参数统一通过 `backend/config.json` 管理，项目启动时自动加载。

- **配置文件**：`backend/config.json`
- **不支持** `.env` 文件或环境变量
- **不支持** 通过前端界面查看或修改配置
- 修改配置后需重启后端服务生效

---

## 一、配置文件完整结构

`backend/config.json` 包含所有参数，示例如下：

```json
{
  "app_name": "Hot Github",
  "debug": true,
  "cors_origins": ["http://localhost:5173", "http://localhost:3000"],
  "download_path": "./github_projects",
  "db_path": "projects.db",
  "log_dir": "logs",
  "github_token": "",
  "github_api_timeout": 30,
  "max_search_results": 5,
  "claude_api_key": "",
  "claude_model": "claude-sonnet-4-6",
  "claude_base_url": "",
  "claude_max_tokens": 8192,
  "claude_cli_timeout": 600,
  "claude_api_timeout": 300,
  "claude_retry_count": 3,
  "analysis_method": "cli",
  "require_login": false,
  "secret_key": "your-secret-key-change-in-production",
  "algorithm": "HS256",
  "access_token_expire_minutes": 1440,
  "users": { "admin": "admin123" },
  "trending_enabled": true,
  "trending_official_limit": 25,
  "trending_starred_limit": 20,
  "trending_categories": {
    "ALL": { "topics": [], "min_stars": 50 },
    "AI": { "topics": ["ai", "machine-learning", "llm"], "min_stars": 100 },
    "实用工具": { "topics": ["cli", "tool", "utility"], "min_stars": 100 },
    "游戏&创意": { "topics": ["game", "creative", "visualization"], "min_stars": 50 },
    "学习资源": { "topics": ["tutorial", "education", "learning"], "min_stars": 50 }
  }
}
```

## 二、参数详细说明

### 2.1 服务器配置

| JSON 字段 | 默认值 | 说明 |
|-----------|--------|------|
| `app_name` | `"GitHub Manager"` | 应用显示名称 |
| `debug` | `true` | 调试模式，开启后端自动重载 |
| `cors_origins` | `["http://localhost:5173", "http://localhost:3000"]` | CORS 允许的前端地址 |

后端监听地址 `0.0.0.0:8000` 硬编码在 `run.py` 中。

### 2.2 路径配置

| JSON 字段 | 默认值 | 说明 |
|-----------|--------|------|
| `download_path` | `"./github_projects"` | GitHub 项目克隆的本地保存位置（相对于项目根目录） |
| `db_path` | `"projects.db"` | SQLite 数据库文件名，实际路径为 `backend/db_path` |
| `log_dir` | `"logs"` | 日志文件保存目录（相对于 `backend/`） |

**实际文件位置**（默认配置下）：

```
./github_projects/              ← 项目下载根目录
./github_projects/01-docs/      ← AI 生成的项目文档
backend/projects.db             ← SQLite 数据库
backend/logs/app.log            ← 应用日志
```

### 2.3 GitHub 配置

| JSON 字段 | 默认值 | 说明 |
|-----------|--------|------|
| `github_token` | `""` | GitHub API 访问令牌（可选） |
| `github_api_timeout` | `30` | GitHub API 请求超时时间（秒） |
| `max_search_results` | `5` | 搜索结果默认数量 |

**获取 Token**：访问 [GitHub Settings → Developer settings → Personal access tokens](https://github.com/settings/tokens)

**不设置 Token 的影响**：匿名访问限速 60 次/小时；设置后提升至 5000 次/小时。

### 2.4 Claude / AI 配置

| JSON 字段 | 默认值 | 说明 |
|-----------|--------|------|
| `claude_api_key` | `""` | Claude API 密钥（可选，CLI 模式下不需要） |
| `claude_model` | `"claude-sonnet-4-6"` | 使用的 AI 模型 |
| `claude_base_url` | `""` | 自定义 API 端点（用于代理或第三方服务） |
| `claude_max_tokens` | `8192` | API 调用最大 token 数 |
| `claude_cli_timeout` | `600` | Claude CLI 调用超时时间（秒） |
| `claude_api_timeout` | `300` | Claude API HTTP 请求超时时间（秒） |
| `claude_retry_count` | `3` | 调用失败重试次数 |
| `analysis_method` | `"cli"` | 分析模式：`"cli"` 或 `"api"` |

**两种分析模式**：

| 模式 | 说明 | 需要配置 |
|------|------|---------|
| `cli`（默认） | 调用本地 `claude` CLI 工具 | 安装 Claude Code CLI |
| `api` | 直接调用 Claude HTTP API | `claude_api_key` |

**可选模型**：

| 模型 ID | 说明 |
|---------|------|
| `claude-sonnet-4-6` | Sonnet 4.6（默认，性价比高） |
| `claude-opus-4-7` | Opus 4.7（最强能力） |
| `claude-haiku-4-5` | Haiku 4.5（最快速度） |
| `glm-5` | 阿里云 GLM-5 |

**API 调用逻辑**：
- 设置了 `claude_base_url` → 请求 `{base_url}/v1/messages`，使用 `Bearer {api_key}` 认证
- 未设置 `claude_base_url` → 请求 `https://api.anthropic.com/v1/messages`，使用 `x-api-key` 头认证

### 2.5 认证配置

| JSON 字段 | 默认值 | 说明 |
|-----------|--------|------|
| `require_login` | `false` | 是否需要登录。`false` 时直接进入系统，无需输入用户名密码 |
| `secret_key` | `"your-secret-key-change-in-production"` | JWT 签名密钥，**生产环境必须修改** |
| `algorithm` | `"HS256"` | JWT 签名算法 |
| `access_token_expire_minutes` | `1440`（24小时） | 登录 Token 过期时间 |
| `users` | `{"admin": "admin123"}` | 登录用户名和密码，**生产环境必须修改** |

> **安全警告**：默认密钥和密码仅用于开发环境，部署前务必修改。设置为 `require_login: true` 可启用登录验证。

### 2.6 Trending 趋势配置

| JSON 字段 | 默认值 | 说明 |
|-----------|--------|------|
| `trending_enabled` | `true` | 是否启用 GitHub Trending 功能 |
| `trending_official_limit` | `25` | 官方热点默认最大获取数量 |
| `trending_starred_limit` | `20` | 高星项目默认最大获取数量 |
| `trending_categories` | 见下方 | 主题分类及筛选条件 |

**trending_categories 结构**：

```json
{
  "分类名称": {
    "topics": ["topic1", "topic2"],
    "min_stars": 100
  }
}
```

- `topics`：GitHub 主题关键词，空数组表示不按主题筛选
- `min_stars`：最低星数过滤

**默认分类**：

| 分类 | 主题 | 最低星数 |
|------|------|---------|
| ALL | 不限 | 50 |
| AI | ai, machine-learning, llm | 100 |
| 实用工具 | cli, tool, utility | 100 |
| 游戏&创意 | game, creative, visualization | 50 |
| 学习资源 | tutorial, education, learning | 50 |

### 2.7 日志配置

| 参数 | 值 | 说明 |
|------|---|------|
| 日志目录 | `log_dir` 配置值 | 相对于 `backend/` |
| 日志文件 | `app.log` | 主日志文件 |
| 轮转策略 | 每日午夜 | `TimedRotatingFileHandler` |
| 保留天数 | 7 天 | 硬编码 |
| 日志级别 | `INFO` | 硬编码 |

## 三、前端配置

前端开发服务器配置硬编码在 `frontend/vite.config.ts` 中：

| 参数 | 值 | 说明 |
|------|---|------|
| 开发服务器端口 | `5173` | 不可配置 |
| API 代理目标 | `http://localhost:8000` | `/api` 路径代理到后端 |
| WebSocket 代理目标 | `ws://localhost:8000` | `/ws` 路径代理到后端 |

---

## 四、配置 API 接口

| 接口 | 方法 | 说明 |
|------|------|------|
| `/api/config/test-github` | POST | 测试 GitHub Token 连通性 |

> 配置的查看和修改已从 API 中移除，所有配置变更需直接编辑 `backend/config.json` 并重启服务。

---

## 五、配置修改方式

1. 编辑 `backend/config.json`
2. 重启后端服务：`cd backend && python run.py`
