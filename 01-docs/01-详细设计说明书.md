## 1. 项目概述

### 1.1 项目定位

GitHub Manager Web（代号：Hot Github）是一个基于 Web 的 GitHub 项目管理平台，旨在帮助用户高效地发现、下载、管理和分析 GitHub 开源项目。该系统集成了 GitHub API、Web 爬虫技术和 AI 文档生成能力，为开发者提供了一个一站式的开源项目管理解决方案。

### 1.2 核心功能

1. **项目搜索与下载**：支持关键词搜索和 URL 直接导入，自动克隆项目到本地
2. **热点项目追踪**：实时抓取 GitHub Trending 页面，记录历史热点数据
3. **项目管理**：本地项目的增删改查、更新同步、标签管理
4. **AI 文档生成**：利用 Claude AI 自动生成项目设计说明书、使用说明书、价值分析报告
5. **定时任务调度**：自动采集热点、下载项目、生成分析报告
6. **用户认证**：可选的登录认证机制

### 1.3 目标用户

- 开源项目研究者
- 技术选型决策者
- 开发者学习者
- AI 辅助编程使用者

### 1.4 技术栈

| 层级 | 技术选型 |
|------|----------|
| 前端框架 | Vue 3 + TypeScript + Vite |
| UI 组件库 | Element Plus |
| 状态管理 | Pinia |
| 路由管理 | Vue Router 5 |
| HTTP 客户端 | Axios |
| 后端框架 | FastAPI (Python) |
| 数据库 | SQLite |
| 任务调度 | APScheduler |
| AI 接口 | Claude API / Claude CLI |
| 爬虫解析 | BeautifulSoup4 |

## 3. 核心功能模块设计

### 3.1 项目搜索与下载模块

#### 3.1.1 功能描述

支持两种搜索方式：
1. **关键词搜索**：调用 GitHub Search API，按 star 数排序返回结果
2. **URL 导入**：直接粘贴 GitHub 仓库 URL，自动解析项目信息

#### 3.1.2 流程设计

```
用户输入 → 判断输入类型
    │
    ├── 关键词 → 调用 GitHub Search API → 返回结果列表
    │
    └── URL → 解析 owner/repo → 调用 GitHub Repo API → 返回单个项目
              │
              ▼
        用户选择项目 → 点击下载
              │
              ▼
        创建下载任务 → 返回 task_id
              │
              ▼
        后台执行 git clone → 实时推送进度 (WebSocket)
              │
              ▼
        下载完成 → 保存项目信息到数据库
```

#### 3.1.3 关键代码实现

**GitHub API 封装** (`backend/app/core/github_api.py`)

```python
class GitHubAPI:
    def search_repositories(self, query: str, limit: int = 5) -> list[dict]:
        """搜索 GitHub 仓库"""
        url = f"{self.BASE_URL}/search/repositories"
        params = {"q": query, "sort": "stars", "order": "desc", "per_page": limit}
        response = self.session.get(url, params=params, timeout=self.timeout)
        return [self._parse_repo(item) for item in data.get('items', [])]

    def get_repository_by_url(self, url: str) -> Optional[dict]:
        """通过 URL 获取仓库信息"""
        owner, repo = self._parse_github_url(url)
        return self.get_repository(owner, repo)
```

**下载器** (`backend/app/core/downloader.py`)

```python
class ProjectDownloader:
    def clone_project(self, clone_url: str, project_name: str) -> tuple[bool, str]:
        """克隆项目到本地"""
        target_path = self.download_path / project_name
        process = subprocess.Popen(
            ['git', 'clone', '--progress', clone_url, str(target_path)],
            stdout=subprocess.PIPE, stderr=subprocess.PIPE, text=True
        )
        # 实时读取进度并回调
        while True:
            line = process.stderr.readline()
            if self.status_callback:
                self.status_callback(f"[git] {line.strip()}")
            if process.poll() is not None:
                break
```

### 3.3 AI 文档生成模块

#### 3.3.1 功能描述

利用 Claude AI 自动分析项目代码，生成三类文档：
1. **设计说明书**：架构设计、技术栈、核心流程
2. **使用说明书**：安装配置、功能使用、最佳实践
3. **价值分析报告**：竞争力分析、市场需求、发展前景

#### 3.3.2 生成流程

```
阶段1: MD 文档生成 (3个并行线程)
    │
    ├── 设计说明书.md
    ├── 使用说明书.md
    └── 价值分析报告.md
          │
          ▼
阶段2: HTML 文档生成 (3个并行线程，依赖阶段1)
    │
    ├── 设计说明书.html
    ├── 使用说明书.html
    └── 价值分析报告.html
          │
          ▼
阶段3: PPTX 文档生成 (3个并行线程，依赖阶段2)
    │
    ├── 设计说明书.pptx
    ├── 使用说明书.pptx
    └── 价值分析报告.pptx
```

#### 3.3.3 AI 调用方式

支持两种方式：

**CLI 模式**（推荐）：
```python
cmd = [claude_path, "-p", "--print", "--dangerously-skip-permissions"]
result = subprocess.run(cmd, input=prompt, capture_output=True, text=True)
```

**API 模式**：
```python
payload = {
    "model": self.model,
    "max_tokens": self.max_tokens,
    "messages": [{"role": "user", "content": prompt}]
}
response = requests.post(api_url, headers=headers, json=payload)
```

#### 3.3.4 重试机制

- 最大重试次数：3 次
- 重试间隔：递增等待（5s × 重试次数）
- 失败记录：保存到 `trending_reports` 表，定时重试

### 3.4 定时任务调度模块

#### 3.4.1 任务配置

| 任务名称 | 执行时间 | 功能 |
|----------|----------|------|
| daily_trending | 每天 22:00 | 采集日热点 |
| weekly_trending | 每周一 22:00 | 采集周热点 |
| monthly_trending | 每月1日 22:00 | 采集月热点 |
| auto_analysis | 凌晨 00:30 | 自动下载和分析 |
| retry_analysis | 每小时整点 | 重试失败任务 |

#### 3.4.2 调度器实现

```python
class TrendingScheduler:
    def _setup_jobs(self):
        self.scheduler.add_job(
            self.collect_daily_trending,
            CronTrigger(hour=22, minute=0),
            id='daily_trending'
        )

    def collect_daily_trending(self):
        scraper = TrendingScraper(self.settings.GITHUB_TOKEN)
        results = scraper.get_trending(since='daily', limit=limit)
        self.db.save_trending_history(results, 'daily', period_value)

        # 自动下载
        if self.settings.HISTORY_TRENDING_AUTO_DOWNLOAD:
            self._auto_download_trending(period_value)
```
