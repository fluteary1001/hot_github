# 历史官方热点 - 手动采集流程

## 一、流程概述

手动采集功能允许用户通过界面触发 GitHub Trending 数据的即时采集，支持三种采集类型：今日热点、本周热点、本月热点。

## 三、详细步骤说明

### 3.1 前端操作（HistoryTrendingPage.vue）

**入口位置**：页面头部右侧操作区

```vue
<el-select v-model="collectType" placeholder="采集类型">
  <el-option label="今日热点" value="daily" />
  <el-option label="本周热点" value="weekly" />
  <el-option label="本月热点" value="monthly" />
</el-select>
<el-button type="primary" @click="handleCollect">
  手动采集
</el-button>
```

**触发函数**：`handleCollect()`

```typescript
async function handleCollect() {
  collecting.value = true
  try {
    const res = await historyApi.manualCollect(collectType.value)
    ElMessage.success(`采集成功，保存了 ${res.data.saved_count} 个项目`)
    await loadTree()  // 刷新树数据
  } catch (error) {
    ElMessage.error('采集失败')
  } finally {
    collecting.value = false
  }
}
```

### 3.4 调度器处理（scheduler.py）

```python
def manual_collect(self, period_type: str = 'daily') -> dict:
    # 1. 创建爬虫实例
    scraper = TrendingScraper(self.settings.GITHUB_TOKEN, timeout=...)
    
    # 2. 计算周期值
    if period_type == 'daily':
        period_value = datetime.now().strftime('%Y-%m-%d')  # 如 2026-05-18
    elif period_type == 'weekly':
        period_value = f"{year}-W{week:02d}"  # 如 2026-W20
    else:
        period_value = datetime.now().strftime('%Y-%m')  # 如 2026-05
    
    # 3. 爬取数据
    results = scraper.get_trending(since=period_type, limit=25)
    
    # 4. 保存到数据库
    saved = self.db.save_trending_history(results, period_type, period_value)
    
    # 5. 自动下载（可选）
    if self.settings.HISTORY_TRENDING_AUTO_DOWNLOAD:
        self._auto_download_trending(period_value)
    
    return {"success": True, "saved_count": saved}
```

### 3.5 爬虫采集（trending_scraper.py）

```python
def get_trending(self, since: str = "daily", language: str = "", limit: int = 25):
    # 构建请求
    url = "https://github.com/trending"
    params = {"since": since}
    
    # 发送请求
    response = self.session.get(url, params=params, timeout=self.timeout)
    
    # 解析 HTML
    soup = BeautifulSoup(response.text, "html.parser")
    articles = soup.select("article.Box-row")
    
    # 提取项目信息
    for article in articles[:limit]:
        project = {
            "name": ...,
            "full_name": ...,
            "html_url": ...,
            "description": ...,
            "stars": ...,
            "stars_today": ...,
            "language": ...,
            "clone_url": ...
        }
        results.append(project)
    
    return results
```

def _auto_download_trending(self, period_value: str):
    # 获取待下载项目

## 四、数据流向

```
用户点击 → API请求 → 爬虫采集 → 数据库存储 → 自动下载 → 返回结果 → 前端展示
```

## 五、关键配置参数

| 参数 | 说明 | 默认值 |
|------|------|--------|
| `TRENDING_OFFICIAL_LIMIT` | 采集项目数量上限 | 25 |
| `GITHUB_TOKEN` | GitHub API Token（可选） | - |
| `GITHUB_API_TIMEOUT` | API请求超时时间 | 30秒 |
| `HISTORY_TRENDING_AUTO_DOWNLOAD` | 是否自动下载项目 | False |
| `DOWNLOAD_PATH` | 项目下载路径 | - |

## 六、采集类型说明

| 类型 | 参数值 | 周期值格式 | 示例 | 数据来源 |
|------|--------|-----------|------|----------|
| 今日热点 | daily | YYYY-MM-DD | 2026-05-18 | GitHub Trending Daily |
| 本周热点 | weekly | YYYY-Wxx | 2026-W20 | GitHub Trending Weekly |
| 本月热点 | monthly | YYYY-MM | 2026-05 | GitHub Trending Monthly |

## 七、数据库表结构

采集数据存储在 `trending_history` 表：

| 字段 | 说明 |
|------|------|
| period_type | 周期类型（daily/weekly/monthly） |
| period_value | 周期值 |
| year, month, week, day | 解析后的时间维度 |
| rank | 排名 |
| project_name | 项目名称 |
| full_name | 完整名称（owner/repo） |
| stars | 总 Star 数 |
| stars_change | Star 变化数 |
| downloaded | 是否已下载（0/1） |

## 八、注意事项

1. **重复采集**：同一周期同一排名的项目会被替换（INSERT OR REPLACE）
2. **网络依赖**：需要能访问 GitHub Trending 页面
3. **下载耗时**：若启用自动下载，下载时间取决于项目大小和网络速度
4. **并发限制**：手动采集会阻塞直到完成，建议不要频繁触发
